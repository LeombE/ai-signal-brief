# Migration Policy

This public repository must be built from clean, public-safe files.

## Core Rules

- Do not copy private Git history.
- Do not copy private generated reports directly.
- Do not copy local automation scripts directly.
- Do not copy secrets, env files, screenshots, attachments, or local-only Codex material.
- Historical reports must be migrated privately into sanitized English canonical records before publication.

## Sanitized Historical Migration

A historical report may be published only after review confirms:

- the report is English-first
- all material claims have source IDs
- no secrets or local paths are present
- no private screenshots or private attachments are included
- third-party material is attributed and not over-quoted
- generated assets are either original, licensed, or replaced

## Public Output Rule

The canonical public artifact is `report.json`. Other outputs should be generated from canonical data rather than hand-maintained as separate sources of truth.
